Interpretation Of Edge Detection Outputs

Here we will discuss how to interpret the outputs produced by experiment 5, i.e., Edge Detection. So, please follow the steps as shown below to get the sample output for interpretation.

1) Select "inputimage" as your experimental image.

                    "inputimage" is a satellite image having 6 bands stored in BIL format as raw image. It has the dimension 801x1401. Since in our discussion we will cover upto its 4th band, so it is necessary to understand that what the first 4 bands of this image represents. The first four bands are as follows: blue, green, red and infra-red respectively.

2) Select band for which you want to find the edge output. Here, we are selecting band "5".

3) Select method of edge detection. Here we will choose "Sobel". (already set as default)

Sobel operator uses two perpendicular masks for detecting horizontal and vertical edges. So, here you have an option to select whether you want only horizontal edges, vertical edges or both. Consequently, you will also have to enter threshold values seperately.

4) Enter "Edge threshold value" as "0.05", "Vertical Threshold" as "0.05" and "Horizontal threshold" as "0.05".

Threshold limits the output to be displayed. By entering 0.05 as threshold you are actually limiting it to show only the magnitude and direction where it is above 5% of highest gray level that can be displayed, which is 255.

5) Click on "Process" button

You will get the output as :

Edge Detected Image

Gradient Magnitude Image

Gradient Direction Image

Here you will get the output at each pixel which corresponds to an edge pixel. You can compare the edge image with the original image produced in any other experiment.

EDGE DETECTORS

The image shown below is band 1 of sample image “quickbird.jpg”.

Prewitt operator

Image below highlights the edges present in the sample band. The bright lines are edges where there is abrupt change in gray values of the pixels. The prewitt operator is employed with threshold 0.5, gradient threshold- vertical 0.2 and gradient threshold- horizontal 0.3. The gradient was computed in both directions. It is to be noted that whenever gradient values fall above 0.5 then those pixels are identified as edges.

The following two figures represent the gradient magnitude at a point and gradient direction respectively. Bright pixels have higher gradient magnitude and gradient direction depicts the direction in which there is more change in gray value. Here first image is gradient magnitude and that follows is gradient direction image.

Robert operator

Threshold value used is 0.2. Values falling above this threshold are identified as edges. The bright lines represent those points where values are higher than 0.2.

Laplacian of Gaussian (LoG)

Threshold chosen to generate edge image below is 0.7 and sigma as 0.5. Laplacian filters are derivative filters and are prone to noise detection. Hence LoG operator smoothens the test image and then applies the laplacian operator. Hence the name LoG.

Canny operator

Threshold =0.2 and sigma=0.5. Please note that in the experiment we enter lower threshold value. The program automatically calculates the higher threshold value (i.e twice the lower threshold). The image shown below represents edges as bright lines. The values that fall above higher threshold are labeled as edges. And those falling below lower threshold are discarded as not edges. Those falling in-between higher and lower are evaluated for nearness. Pixels with weak connectivity are not joined whereas pixels with higher chance of connectivity are connected.
