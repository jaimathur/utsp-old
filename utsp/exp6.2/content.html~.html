Accuracy Assessment

Back To Experiment List

Accuracy Assessment is estimated using ground truth i.e., set of points whose class is known. These points are known as test points. Test points should not be used during the classifier training. Test points should be in sufficient number for the accuracy assessment to be significant.

Choice of Test Points

Test points should not be taken in the immediate neighborhood of training data points. High spatial dependence would bias the assessment in such a case. Based on the accuracy of classification of the test data points, various quantitative estimates are defined. The degree of agreement between the classifier output and the ground truth is considered.

Confusion Matrix

Error or confusion matrix is often deployed to derive measures of accuracy. It is used to derive the overall accuracy as well as user’s accuracy and producer’s accuracy.

 

Measures of Accuracy from Confusion Matrix

Overall accuracy, user's accuracy and producer's accuracy can be computed using confusion matrix as:

Overall Accuracy

It is calculated as number of test samples correctly classified / Total number of test samples. i.e., Sum of diagonal elements / Sum of rows, in confusion matrix.

Producer’s Accuracy (Omission errors)

It is calculated as no. of samples of a class correctly classified / No. of samples of the class.

User’s Accuracy (Commission errors)

It is calculated as no. of samples that actually belong to a class / No. of samples assigned to that class.

Kappa coefficient

Kappa coefficient is defined as



Ci+ denotes the sum of the ith row elements; C+i denotes the sum of the ith column elements κ is interpreted as the index of correct classification after adjusting for chance agreement between the true and computed classes.

Interpretation of Kappa Coefficient

Kappa coefficient is used to compare the degree of consensus between raters (inspectors) in, for example, Measurement Systems Analysis. The degree of chance agreement between the raters and the measurement system is considered in the rating to assess the actual degree of agreement.

Suppose 150 samples are independently verified and the following determinations are made:

The chance values are produced by computing:

(row total x col. total) / Total values

130 comes from the first table, 100.92 comes from the chance table, 150 is the total number of samples. Chance agreement is deducted from actual

agreement to estimate the degree of agreement between the system (classifier) and the inspector (ground truth).

Interpretation Of Accuracy Assessment Outputs

Error Matrix



The table above is called an “error matrix” or “confusion matrix” generated using test samples taken from sam1.jpg image and tested on the classified image of the same using Maximum Likelihood Classification technique.

It depicts how the test samples of different classes/features have been classified by a technique (here MLC technique). So, looking at the table you can infer that there are 6 classes in the classified image. It can also be inferred that

[1] Total number of samples taken for each class

For example from the “total” column a total of 270 samples have been taken as test samples for class1, 306 for class6, 189 for class5 and so on

[2] The number of samples correctly classified and incorrectly classified by the classification technique.

For example consider class 4, out of total 1222 samples 1199 have been correctly classified and 21 have been incorrectly classified as class 3.

[3] Number of samples out of the Total (i.e. 2270) that have been classified in a particular class. For example consider class 3, a total of 318 (look at row for Total) samples have been classified as class 3. But actually we have taken a total of 214 samples as belonging to class 3. So, it is to understood that additional samples (belonging to other classes) have been misclassified as class 3.







The overall accuracy tells about the percentage of samples that were correctly classified. Indirectly it denotes the reliability of a classification technique.

Producer’s accuracy for a class is also called as “error of commission”. It indicates the percentage of ground-truth data that has been correctly classified for each class.

User’s accuracy comes into picture when someone else takes your classified image and tests it with his/her own ground-truth data (i.e. test samples). It is also named as “error of omission”.

Kappa coefficient is a more accurate way of evaluating the classification capability of a technique.

FAQs

Q 1. What does user requires to have to be able to use virtual satellite image processing and analysis lab?

Ans : Virtual Satellite image processing and analysis lab setup only requires internet connection and a web browser at the student's end to connect to the lab. The lab is self-contained, with sample images, aim, theory procedure, actual experiment and interpretation of the results. It is highly recommended to use v-SIPAL in mozilla firefox or google chrome and have adobe flash player installed in you PC.

Q 2. Which image formats are supported?

Ans : v-SIPAL supports all basic image formats like JPEG, BMP, TIFF, geoTIFF. In addition to these, raw satellite images can also be used with header file having an extension (.hdr)

Q 3. Are sample images there for users?

Ans : Each user is provided with two sample images : Sample 1 is a raw satellite image with 6 bands, Sample 2 is a jpeg file with 3 bands.

Q 4. Is there any restriction on image size to upload?

Ans : One can upload images of any size. But it is highly recommended to not upload images having dimensions more than 2048x2048.

Q 5. Can user download the output images?

Ans : Yes. User can download output images and histograms by just right clicking on them and then clicking on "Save image as".
