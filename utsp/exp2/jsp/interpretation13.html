Interpretation Of  Accuracy Assessment Outputs

Error Matrix

The table above is called an “error matrix” or “confusion matrix” generated using test samples taken from sam1.jpg image and tested on the classified image of the same using Maximum Likelihood Classification technique.

It depicts how the test samples of different classes/features have been classified by a technique (here MLC technique). So, looking at the table you can infer that there are 6 classes in the classified image. It can also be inferred that

[1] Total number of samples taken for each class

For example from the “total” column a total of 270 samples have been taken as test samples for class1, 306 for class6, 189 for class5 and so on

[2] The number of samples correctly classified and incorrectly classified by the classification technique.

For example consider class 4, out of total 1222 samples 1199 have been correctly classified and 21 have been incorrectly classified as class 3.

[3] Number of samples out of the Total (i.e. 2270) that have been classified in a particular class. For example consider class 3, a total of 318 (look at row for Total) samples have been classified as class 3. But actually we have taken a total of 214 samples as belonging to class 3. So, it is to understood that additional samples (belonging to other classes) have been misclassified as class 3.

The overall accuracy tells about the percentage of samples that were correctly classified. Indirectly it denotes the reliability of a classification technique.

Producer’s accuracy for a class is also called as “error of commission”. It indicates the percentage of ground-truth data that has been correctly classified for each class.

User’s accuracy comes into picture when someone else takes your classified image and tests it with his/her own ground-truth data (i.e. test samples). It is also named as “error of omission”.

Kappa coefficient is a more accurate way of evaluating the classification capability of a technique.
